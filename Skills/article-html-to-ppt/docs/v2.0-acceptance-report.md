# MeowClaw PPTSmith v2.0 Acceptance Report

**Acceptance date:** 2026-07-19 (Asia/Shanghai)

**Verdict:** Engineering-complete and **Standard production-ready on this verified environment**.

**Canonical Builder:** `python_pptx` 1.0.2.

**Portability Builder:** `pptxgenjs` 4.0.1, evidence only—not a second canonical final.

**Premium:** **not final**; externally blocked until a verified PowerPoint/LibreOffice renderer supplies render/readback evidence.

## Accepted scope

The v2 acceptance fixture is one nine-slide article deck covering cover, judgment, evidence, editable table, native chart, simple process, comparison, implementation plan, and closing. Fresh isolated Standard runs proved that `python_pptx` and PptxGenJS consume byte-identical PPT IR, Style Contract, and requirements contracts. The source Delivery Plan has builder-neutral object/route semantics, and both authoritative resolved plans retain the same `selected_route` values. Their capability-sensitive resolver metadata differs: Python records one resolver-level fallback decision for S05 `coverage-chart` while retaining `selected_route: "native_chart"`, with reason codes `DELIVERY_BUILDER_PARTIAL_SUPPORT` and `DELIVERY_FALLBACK_SELECTED` and Delivery Plan `fallback_count: 1`; PptxGenJS records `decision: "selected"` and Delivery Plan `fallback_count: 0`.

The unique committed package is `deliverables/pptsmith-v2-acceptance/` in the GitHub source repository. Its README is the canonical command transcript and provenance record. `python_pptx` is the sole primary production deck. The PptxGenJS deck is labeled portability evidence throughout. The standalone ClawHub package does not duplicate this repository-level evidence bundle.

## Documentation claim

The public/documented version is `2.0.0`. The ClawHub slug, installed OpenClaw route, and directory remain `article-html-to-ppt` for compatibility and update continuity; the public brand identity is MeowClaw PPTSmith, with `meowclaw-pptsmith` as its brand alias.

This readiness claim is intentionally narrow: Standard production is locally accepted in the verified environment above. It is not a claim that Premium is final, that rendering was visually approved, or that every external Office environment is identical.

## Prominent limitations

1. **Premium final acceptance remains externally blocked.** No verified PowerPoint/LibreOffice renderer exists on this host, so no full-render, readback, or rubric-final evidence exists.
2. PptxGenJS retains 11 warning-only decorative empty-text-box findings.
3. Required style fonts are missing on this host; consumers may see substitutions.
4. PptxGenJS process arrows are editable shape primitives rather than OOXML connector objects, so the generic connector counter reports zero for that deck.
5. `officecli`, `html_svg`, animation, transitions, narration, timing, video, publishing, and v2.1 are outside this acceptance.
