# v1.5 / v2.0 Closeout Checklist

This checklist keeps the version claim honest.

## Current Version

Current public skill version: `1.2.0`.

Stages 1-10 are complete. They established contract-first planning, schemas, validators, Diagram IR, package inspection, render adapters, verification orchestration, benchmark fixtures, production profiles, capability probe, builder selection, trusted delivery manifests, and migration docs.

## v1.5 Exit Criteria

v1.5 is complete when the skill can run a truthful local production chain for ordinary decks:

- `python_pptx` can create a real PPTX for native text, cards, and simple native tables.
- `scripts/build_deck.py` writes a valid `build-manifest.json`.
- The generated PPTX passes package inspection.
- Capability Probe, Delivery Plan, Builder selection, Build Manifest, Verify/Inspect, and Delivery Manifest can be run as separate CLI gates.
- Standard profile can complete without pretending that unavailable renderers exist.

## v2.0 Exit Criteria

v2.0 is complete only when multi-builder production is stable:

- At least two real builders can produce PPTX from the same PPT IR and Delivery Plan.
- Native-required text, cards, tables, charts, and simple diagrams stay editable.
- Complex diagrams either build as native/hybrid with disclosed fallbacks or stop before overclaiming.
- Fast, Standard, and Premium profile fixtures all run through end-to-end regression.
- Premium has real full-render evidence, readback/inspection, zero QA errors, rubric pass, and disclosed fallbacks before `final`.
- Builder route deviations are recorded in Build Manifest and detected by validation.
- One real 8-10 slide article deck is generated, inspected, packaged, and reviewed as the canonical v2 acceptance sample.

## Current Known Gaps

- `pptxgenjs`, `officecli`, and `html_svg` still have contract adapters, not production build runtimes.
- `python_pptx` is now the first minimal runtime builder, but chart and complex diagram output remain fallback/placeholder territory.
- This machine still lacks a real PowerPoint/LibreOffice renderer, so Premium cannot reach `final` here.
- End-to-end sample deck generation is not yet a single command.
