# PPT Production Artifact Checklist

Use for non-trivial decks before final handoff.

- [ ] `content_analysis.md` exists and includes thesis, audience, evidence, risks, reusable terms.
- [ ] `storyboard.md` exists and includes slide sequence, judgment titles, audience action, source coverage.
- [ ] `slide_manifest.json` exists and includes expression mode, page archetype, primary anchor, editable core, and visual component plans.
- [ ] `style_contract.json` exists and defines typography, color, layout primitives, component grammar, forbidden drift.
- [ ] `detailed_design_spec.md` or `design_spec.json` exists.
- [ ] `spec_implementation_plan.json` exists and includes visual priority, deletion rules, noise budget, object roles.
- [ ] `visual_reference/` exists when polish matters, with at least cover and one representative content page.
- [ ] `deck.pptx` exists.
- [ ] `verification-report.md` exists with object/media counts, render/readback route, known limits.
- [ ] `delivery-manifest.json` exists with canonical final paths and final status.
