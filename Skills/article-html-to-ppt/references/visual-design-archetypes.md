# Visual Design Archetypes

Use this reference after reading the article and before applying a brand visual system. Pick the archetype that fits the article's content, but keep it inside a stable brand-consistent deck system.

## pressure-fracture-editorial

Use when the article argues that something polished breaks under real-world pressure.

Fits:

- AI demo vs real work
- product illusion
- workflow stress test
- hidden fragility

Professional treatment:

- Use pressure/fracture as a restrained motif, not as a different background on every page.
- Prefer one dominant background family with coral fracture accents.
- Keep title position, footer, caption style, and brand marker consistent.

Visual language:

- contrast smooth surface vs fracture / pressure / weight
- strong single accent for rupture
- diagrams show pressure paths, breaking points, or layers of resistance

Layouts:

- cover with large title and one pressure visual
- contrast pages: demo path vs real work resistance
- evidence pages: reconstructed UI with explicit label
- framework page: layered model or force diagram

Avoid:

- generic dashboard cards
- dense bullets on every slide
- decorative crack textures that reduce readability
- frequent dark/light background switching without chapter logic

## clean-tutorial-checklist

Use when the article teaches steps, procedures, troubleshooting, or implementation choices.

Visual language:

- light paper background
- numbered steps
- pass/fail markers
- simple tables and checklists

Layouts:

- step cards
- before/after examples
- compact decision tree
- error/fix comparison

Avoid:

- dark cinematic mood when clarity matters
- too many decorative icons

## signal-news-briefing

Use for news, industry observation, monthly events, policy updates, market movements.

Visual language:

- newsroom / signal / timeline
- one main event per slide
- facts vs implications vs uncertainties separated clearly

Layouts:

- event cards
- timeline
- signal map
- impact matrix

Avoid:

- ten-news collage
- all-blue glowing tech wallpaper

## executive-thesis-briefing

Use for business, product, strategy, investment, or decision briefings.

Visual language:

- calm hierarchy
- clear claims
- few colors
- tables, matrices, risk/decision boxes

Layouts:

- claim + evidence
- decision matrix
- tradeoff table
- roadmap or operating model

Avoid:

- overly literary cover mood
- decorative visuals that obscure the decision

## narrative-essay-stage

Use for reflective essays, personal voice, storytelling, or keynote-style talks.

Visual language:

- large typographic statements
- warm low-saturation mood
- fewer bullets
- images or abstract scenes carry emotion

Layouts:

- quote slides
- chapter dividers
- slow reveal sequence
- closing statement slide

Avoid:

- busy framework diagrams on every page
- too much operational UI language

## framework-map

Use when the article's core value is a reusable model, checklist, taxonomy, or mental map.

Visual language:

- central diagram
- layered or modular structure
- stable labels
- accent colors mapped to meaning

Layouts:

- one overview model
- one slide per layer/module
- summary checklist

Avoid:

- making the framework a tiny side element
- using colors with no semantic mapping
