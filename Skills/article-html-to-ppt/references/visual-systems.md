# Visual Systems Reference

Use this file after deriving content-fit direction. These systems define the brand frame; article-specific design must stay inside them.

## meowclaw-lab

Use for MeowClawLab, 夜猫子弦月, AI Agent practice, workflow transformation, tool/product judgment, and article-to-deck tasks.

Palette:

| Token | Hex | Use |
| --- | --- | --- |
| moon-ink | #14161F | deep text / formal dark background |
| lunar-slate | #2A3142 | secondary panels and muted text |
| claw-coral | #FF6B5F | rupture, warning, action, core tension |
| copper-glow | #C9824B | warm narrative emphasis |
| mist-white | #F6F4EF | formal light background and text surface |
| terminal-green | #2DBE7E | success and verified state |
| warning-amber | #F2B84B | risk and cost |

Formal deck rules:

- Default to formal-light body pages for readability unless the user asks for dark keynote style.
- Use MeowClaw colors semantically, not all at once.
- Keep a visible but restrained `MeowClawLab` mark or approved brand text on content pages.
- Keep footer/source/caption pattern consistent.
- Keep title position and spacing stable.
- Mark reconstructed UI as reconstructed illustration.
- Respect source rights and usage boundaries before public sharing.
- Prefer article-specific editorial diagrams over decorative dashboards.

Avoid:

- full-screen generic blue-purple AI gradient
- every page using a different background system
- over-cute imagery
- decorative dashboards that do not explain the argument

## pitfall-guide

Use for AI Agent tutorials, checklists, configuration, troubleshooting, service entry decks.

Palette:

| Token | Hex | Use |
| --- | --- | --- |
| paper-white | #FAFAF7 | tutorial background |
| guide-navy | #1E2A3A | body text |
| fix-green | #2DBE7E | pass / correct / recommended |
| bug-red | #D94A4A | error / anti-pattern |
| note-blue | #4A90E2 | parameter / explanation |

Rules:

- Prefer shallow hierarchy and checklist structure.
- Separate error and fix colors clearly.
- Keep each slide to one problem or one step.
- Maintain consistent footer/caption and step-number patterns.

## nightly-report

Use for 猫爪夜报, AI news, industry observation, monthly event maps.

Palette:

| Token | Hex | Use |
| --- | --- | --- |
| news-black | #101218 | night report background |
| signal-blue | #3E7BFA | tech/model signal |
| alert-gold | #F5C542 | major event / alert |
| paper-gray | #E8E8E2 | readable card surface |
| redline | #E85A4F | risk, regulation, warning |

Rules:

- Separate fact, implication, and uncertainty.
- Prefer event timelines, signal matrices, and source-backed bullets.
- Do not make every item visually equal when importance differs.
- Keep recurring report header/footer rhythm.
