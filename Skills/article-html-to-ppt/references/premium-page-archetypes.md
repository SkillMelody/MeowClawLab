# Premium Page Archetypes

Choose one primary archetype per slide. If two archetypes compete, split the slide or demote one.

## Core Archetypes

- `executive-cover`: title, thesis, context, restrained brand marker.
- `section-opener`: transition statement plus one memorable visual or framing line.
- `claim-evidence`: judgment title plus proof component.
- `metric-proof`: one to three key metrics with interpretation.
- `framework-map`: grouped conceptual framework or 2x2/2x3 system.
- `process-lane`: sequence, swimlane, workflow, or before/after operating path.
- `system-architecture`: layered architecture, ownership boundary, data/control path.
- `ecosystem-map`: actors, incentives, dependencies, value exchange.
- `decision-matrix`: options, criteria, scores, recommendation.
- `risk-register`: risk, signal, impact, mitigation, owner.
- `observability-dashboard`: KPI/health panel where dashboard reading is the actual slide job.
- `implementation-roadmap`: phases, milestones, dependencies, decisions.
- `comparison-contrast`: two or three alternatives or states.
- `editorial-scene`: visual-first metaphor, persona scene, or IP illustration.
- `closing-standard`: conclusion, ask, next step, or decision checkpoint.

## Archetype Rules

- Archetype follows audience task, not template preference.
- Relationship-heavy archetypes must pass Expression Mode Gate.
- Tables and dashboards require a reason to be dense.
- Framework maps should not hide dependencies that matter.
- Technical architecture pages need ownership/trust boundaries when relevant.
